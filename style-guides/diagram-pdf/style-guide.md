# Diagram Style Guide

Style rules for standalone TikZ diagrams generated by the `diagram-pdf` skill. Colors, typography, and node conventions match the Beamer slide style guide so diagrams and slides in the same project share a visual identity. The PPTX style guide uses the same accent palette but a slightly lighter body-text color (`#4D4D4D` vs. `#3A3A3A`); the rationale is documented in the PPTX style guide.

## Document Class and Packages

```latex
\documentclass[border=16pt]{standalone}
\usepackage{tikz}
\usetikzlibrary{arrows.meta,positioning,calc,decorations.pathreplacing,shapes.geometric,fit}
```

All diagrams use the `standalone` class with 16pt border padding. This produces a tightly cropped PDF sized to the diagram content.

## Color Palette

Identical to the Beamer slide palette. All colors must be defined in every diagram even if unused, for forward compatibility.

```latex
\definecolor{SlateNavy}{HTML}{1B2A4A}
\definecolor{DeepTeal}{HTML}{0D7377}
\definecolor{CyanBlue}{HTML}{0077B6}
\definecolor{DustyPlum}{HTML}{9B5978}
\definecolor{BurntOrange}{HTML}{BF5700}
\definecolor{WarmAmber}{HTML}{E8913A}
\definecolor{AccentGreen}{HTML}{27AE60}
\definecolor{AccentRed}{HTML}{C0392B}
\definecolor{MedGray}{HTML}{B0AFA8}
\definecolor{CharText}{HTML}{3A3A3A}
\definecolor{PaleBlue}{HTML}{E8F0F8}
\definecolor{LightGray}{HTML}{F0EFEC}
```

### Semantic Color Assignments

| Role | Color | Fill | Text |
|------|-------|------|------|
| Primary node (core step) | SlateNavy | solid | white |
| Secondary node (engine, conversion) | DeepTeal | solid | white |
| Optional node | CyanBlue | solid | white |
| Alternate entry / side path | DustyPlum | solid | white |
| User input / decision | WarmAmber | 15% fill | CharText |
| Deliverable output | AccentGreen | 12% fill | CharText |
| Shared context / reference | DustyPlum | 12% fill | CharText |
| Milestone / decision point | MedGray border | LightGray fill | CharText |
| Orchestrator / meta-label | SlateNavy border | 8% fill | SlateNavy |
| Negative / warning | AccentRed | 12% fill | CharText |
| Highlight / emphasis | BurntOrange | 15% fill | CharText |

For thematic diagrams, assign theme clusters colors from this rotation: SlateNavy, DeepTeal, CyanBlue, DustyPlum, BurntOrange, AccentGreen. Use the solid fill for cluster headers and 10-15% fill for cluster body areas.

## Typography

All diagrams use `\sffamily` (sans-serif) as the base font.

| Element | Font specification |
|---------|-------------------|
| Diagram title | `\sffamily\Large\bfseries`, SlateNavy |
| Subtitle | `\sffamily\small`, MedGray |
| Primary node text | `\sffamily\small\bfseries`, white |
| Secondary/optional node text | `\sffamily\small\bfseries`, white |
| Side entry node text | `\sffamily\footnotesize\bfseries`, white |
| Node subtitle (inside node) | `\scriptsize\color{white!80}` |
| User input text | `\sffamily\footnotesize`, CharText |
| Output text | `\sffamily\footnotesize`, CharText |
| Output sub-label | `\scriptsize`, CharText |
| Arrow label | `\sffamily\scriptsize`, CharText, fill=white |
| Annotation / side note | `\sffamily\scriptsize`, context-appropriate color |

### Thematic Layout Nodes

```latex
themeheader/.style={
    draw=#1, fill=#1, text=white,
    rounded corners=4pt, minimum width=4.0cm, minimum height=0.8cm,
    align=center, font=\sffamily\small\bfseries, inner sep=5pt
},
themebody/.style={
    draw=#1, fill=#1!8, text=CharText,
    rounded corners=4pt, minimum width=4.0cm,
    align=left, font=\sffamily\footnotesize, inner sep=6pt,
    text width=3.6cm, anchor=north
},
```

**Grid layout rule:** When two or more `themebody` nodes appear together in a grid, flow, or comparison layout, every body must have `minimum height` set to a common explicit value. Pick a value at least equal to the tallest cluster's natural content height. This keeps cluster boxes visually aligned regardless of per-cluster content volume. Relying on content alone to set height produces uneven cluster boxes, which audit check #5 flags as a defect.

```latex
thesisnode/.style={
    draw=SlateNavy, fill=SlateNavy!10, text=SlateNavy,
    rounded corners=6pt, minimum width=6.0cm, minimum height=1.0cm,
    align=center, font=\sffamily\small\bfseries,
    inner sep=8pt, very thick
},
```


### Arrow Label Style

```latex
lbl/.style={
    font=\sffamily\scriptsize, text=CharText, fill=white, inner sep=1pt
}
```

Labels are placed with `node[lbl]` on the arrow path. Position using `midway`, `pos=0.2`, `pos=0.8`, or explicit placement. Use `above=2pt` or `below=2pt` for offset.

## Layout Rules

### Connector-Node Conflict Resolution

When an arrow path crosses through an unrelated filled node (a node that the arrow does not connect to), this is a layout conflict. Do NOT fix by placing the arrow on a background layer. Instead, reposition the source or destination element so the arrow path passes through clear space.

Common fix: shift the connected element to a y-position between pipeline steps (for pipeline layouts) so its arrow enters the target diagonally via `north west`/`north east` anchors, clearing the content boxes on either side.

An arrow should only render behind another element when they are semantically related (e.g., a connector entering a box it feeds into).

### Spacing Constants

| Measurement | Value |
|-------------|-------|
| Vertical gap between pipeline steps | 2.0-2.3cm |
| Vertical gap with milestone between steps | 1.8cm per element |
| Horizontal offset for content boxes (inputs) | 3.6cm from center |
| Horizontal offset for content boxes (outputs) | -3.6cm from center |
| Horizontal position for left side entries | -8.5cm |
| Horizontal position for right side entries | 10.5cm |
| Minimum gap between node edge and arrow label | 3pt |
| yshift for paired flanking nodes | 16pt / -16pt |
| Arrow yshift from spine to paired nodes | 6pt / -6pt |
| Title to first element gap | 1.5-2.0cm |
| Last element to legend gap | 2.0cm |

### Side Entry Positioning

Side entries must be positioned at y-coordinates between main pipeline steps, not at the same y-coordinate as the step they connect to. This ensures their connecting arrows pass through clear space above or below the content boxes (inputs and outputs) that sit beside each pipeline step.

Use `north west`/`north east` anchors on the target pipeline step to create diagonal arrow paths that clear the content boxes.

### Feedback Arrow Routing

Feedback arrows (arrows that route backward in the pipeline, connecting a downstream element to an upstream element) must not cross through unrelated nodes. Route them through safe corridors outside the diagram's element footprint.

**Routing rules:**

1. Feedback arrows use only horizontal and vertical segments with `rounded corners=8pt`.
2. Vertical segments run through corridors at x-coordinates beyond the leftmost or rightmost element edge plus 0.5cm margin. Corridor positions are derived from Phase 1 bounding box extraction (see the skill's SKILL.md Step 3).
3. Horizontal segments run at y-coordinates verified clear of all elements.
4. Use TikZ calc `let` syntax to reference node positions dynamically:

```latex
\draw[arrowfeed, rounded corners=8pt]
  let \p1 = (source.anchor), \p2 = (target.anchor) in
  (\p1) -- (corridor_x, \y1) -- (corridor_x, \y2) -- (\p2);
```

5. Labels on feedback arrows are placed at the corridor edge using absolute coordinates verified clear of all elements, with `anchor=west` (right corridor) or `anchor=east` (left corridor).
6. The `arrowfeed` style is used for all feedback arrows. The `arrowloop` style is an alias.

## Legend Format

Every diagram includes a legend at the bottom. The legend uses a two-column layout:

**Left column:** Node type swatches with labels (colored rectangles, 0.5cm x 0.3cm, rounded corners=2pt)

**Right column:** Input/output type swatches with labels

**Arrow rows:** Short arrow samples with labels (main flow, optional path, alternate entry, feedback loop as applicable)

Legend title: `\sffamily\small\bfseries`, CharText, `anchor=north west`.
Legend labels: `\sffamily\footnotesize`, CharText, `anchor=west`.
Row spacing: 0.5cm vertical gap between rows.

## Title and Subtitle

Every diagram begins with a title and subtitle positioned at the top center:

```latex
\node[font=\sffamily\Large\bfseries,text=SlateNavy] at (0,1.5) {Diagram Title};
\node[font=\sffamily\small,text=MedGray] at (0,0.8) {Subtitle or description};
```

Adjust y-coordinates based on the diagram's vertical extent. The title should have at least 1.0cm clearance above the first diagram element.

## Compilation

Compile with `pdflatex -interaction=nonstopmode`. Single pass is sufficient (no bibliography or cross-references). If working in a cloud-synced directory (Dropbox, iCloud Drive, OneDrive), compile to a temporary filename first, then copy to the final location.
