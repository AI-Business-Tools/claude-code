# Beamer Slide Style Guide

Use this specification to generate Beamer presentations with a consistent, original design. Follow every detail exactly.

**Background:** White. **Footer:** None.

---

## Document Class

```latex
\documentclass[aspectratio=169,10pt]{beamer}
```

- **Aspect ratio:** 16:9
- **Base font size:** 10pt

## Beamer Theme Configuration

Start from `default` theme and override everything:

```latex
\usetheme{default}
\usecolortheme{default}

% Canvas and text
\setbeamercolor{background canvas}{bg=white}
\setbeamercolor{normal text}{fg=CharText}

% Titles
\setbeamercolor{frametitle}{fg=CharText}
\setbeamercolor{title}{fg=SlateNavy}
\setbeamercolor{subtitle}{fg=DeepTeal}
\setbeamercolor{author}{fg=CharText}
\setbeamercolor{date}{fg=MedGray}

% Structure
\setbeamercolor{structure}{fg=DeepTeal}

% Blocks
\setbeamercolor{block title}{bg=SlateNavy,fg=white}
\setbeamercolor{block body}{bg=PaleBlue,fg=CharText}
\setbeamercolor{block title alerted}{bg=WarmAmber,fg=white}
\setbeamercolor{block body alerted}{bg=PaleBlue,fg=CharText}

% Lists
\setbeamercolor{itemize item}{fg=DeepTeal}
\setbeamercolor{itemize subitem}{fg=CyanBlue}
\setbeamercolor{enumerate item}{fg=DeepTeal}
```

## Font Configuration

```latex
\setbeamerfont{frametitle}{size=\Large,series=\bfseries}
\setbeamerfont{title}{size=\LARGE,series=\bfseries}
\setbeamerfont{subtitle}{size=\normalsize}
\setbeamerfont{author}{size=\small}
\setbeamerfont{date}{size=\small}
\setbeamerfont{block title}{size=\normalsize,series=\bfseries}
```

### Minimum Font Sizes for TikZ Content

When creating TikZ diagrams, charts, and visual elements on slides, use these minimum sizes:

| Size | LaTeX command | Use for |
|------|--------------|---------|
| ~9pt | `\small` | Default for TikZ node content, info cards, calculation chains, featured quotes |
| ~8pt | `\footnotesize` | Chart axis labels, table body text, secondary labels |
| ~7pt | `\scriptsize` | Chart tick labels, citation text (`\sourcecite`), table footnotes only |
| ~5pt | `\tiny` | Never use for any visible element |

**Principle:** If content does not fit at the target font size, reduce content rather than reducing font size. Split the slide, remove a bullet, or condense phrasing before dropping below `\small` for primary content.

## Templates

### Navigation

```latex
\setbeamertemplate{navigation symbols}{}
\setbeamertemplate{itemize items}[circle]
\setbeamertemplate{enumerate items}[default]

\setlength{\leftmargini}{1.5em}
\setlength{\leftmarginii}{1.2em}
```

The `\leftmargini` and `\leftmarginii` values produce a visible hanging indent on wrapped bullet items at 10pt base font size. Continuation lines align under the text start, not under the bullet marker.

### Custom Frametitle

Plain title with no graphical enhancements (no underlines, horizontal rules, or decorative elements):

```latex
\setbeamertemplate{frametitle}{%
  \vskip6pt%
  \begin{beamercolorbox}[wd=\paperwidth,leftskip=18pt,rightskip=18pt]{frametitle}%
    \usebeamerfont{frametitle}\insertframetitle%
  \end{beamercolorbox}%
  \vskip4pt%
}
```

### Content Margins

Set explicit left and right margins to match the frametitle's 18pt indent. Without this, content inside columns can drift further left or right than the title:

```latex
\setbeamersize{text margin left=18pt, text margin right=18pt}
```

**Box-bullet alignment:** When a slide uses colored boxes (TikZ nodes or `\colorbox`) with bullet lists below them (as in a three-column layout), the itemize environment adds its own left margin (`\leftmargini`). This can cause bullets to appear indented further right than the box above them. To align bullets directly under boxes, either:
- Set the box's `text width` to `\columnwidth` (full column width) so the box spans the same area as the bullets, or
- Reduce the itemize indent: `\begin{itemize}[leftmargin=1em]` to bring bullets closer to the column's left edge

The goal is for bullet text to start at the same left position as the box's visible left edge.

### Footer Zone

The bottom 8mm of the slide is reserved as a footer zone. No content should appear in this zone. The `\sourcecite{}` macro positions citations just above it.

```latex
\setbeamertemplate{footline}{%
  \hspace{12pt}\vskip8mm%
}
```

## Custom Macros

### `\highlight{text}` (Inline Highlight)

Small inline badge with SlateNavy background and white bold text:

```latex
\newcommand{\highlight}[1]{\colorbox{SlateNavy}{\textcolor{white}{\small\textbf{#1}}}}
```

**Scope restriction:** Never use `\highlight{}` inside `itemize`, `enumerate`, or any running text. The badge disrupts the visual rhythm of bullet text and creates contrast issues. Reserve `\highlight{}` exclusively for standalone labels in TikZ diagrams (e.g., badge annotations on flowcharts or diagram elements).

For inline emphasis within bullet text, use bold colored text instead:

```latex
% WRONG: Never use \highlight inside bullets
\item Missing piece: \highlight{actual spending data} showing how...

% RIGHT: Bold colored text for inline emphasis
\item Missing piece: \textbf{\color{DeepTeal}actual spending data} showing how...
```

### `\sourcecite{text}` (Source Citation)

Places a source citation at a fixed position near the bottom-right of the slide, just above the footer margin. Uses a TikZ overlay for absolute positioning so the citation is always in the same spot regardless of content above it. MedGray scriptsize upright, right-aligned:

```latex
\newcommand{\sourcecite}[1]{%
  \begin{tikzpicture}[remember picture,overlay]
    \node[anchor=south east,text=MedGray,font=\scriptsize,inner sep=0pt,
      text width=\dimexpr\paperwidth-36pt\relax,align=right]
      at ([xshift=-18pt,yshift=10mm]current page.south east) {#1};
  \end{tikzpicture}%
}
```

The TikZ overlay positions the citation 18pt from the right edge and 10mm above the bottom of the slide (2mm above the 8mm footer zone). This is independent of the frame's text area, so it always sits just above the footer zone. The citation does not participate in the frame's vertical layout and cannot be pushed around by `\vfill` or content above it.

**Right-alignment is critical.** The node uses `align=right` to right-justify text within the box. Do NOT use `\RaggedLeft` or `\raggedleft` inside TikZ nodes; TikZ ignores LaTeX paragraph commands and only respects the `align` key. The `anchor=south east` positions the node's right edge at the right margin, and `align=right` ensures multi-line citations are flush-right.

**Clearance:** The `\sourcecite` overlay occupies approximately the bottom 10mm of the slide. Any TikZ content or frame text that extends below `y = -2.0` (relative to a center-anchored tikzpicture) or below approximately 12mm from the page bottom will collide with citation text. On slides using `\sourcecite`, verify that no content enters this zone. If a TikZ diagram must extend low, either shorten the citation text, move the citation to `\note{}`, or increase the `yshift` value.

**Maximum content height rule:** On any slide that uses `\sourcecite{}`, body content (bullets, tables, TikZ diagrams) must not exceed approximately 80% of the slide height. When content is dense enough to risk collision, reduce the content (fewer bullets, tighter wording, split across slides) rather than shrinking font sizes or removing the citation. This is the most common source of visual defects in Beamer decks; the quality audit checks every slide individually for footer zone encroachment.

**Important:** The citation font is `\scriptsize` (upright, not italic). Titles within the citation use `\textit{}` to produce visible italicization. If the entire citation were italic (`\itshape`), then `\textit{}` would have no visible effect and the title would not be distinguished from the rest of the citation.

### Citation Format

Source citations follow Chicago Author-Date. Every citation must have all four components: **Author. Year. Title. Publication.**

- **Single author:** `Last, First. Year. \textit{Title.} Publication.`
- **Two authors:** `Last, First, and First Last. Year. \textit{Title.} Publication.`
- **Three or more:** `Last, First, et al. Year. \textit{Title.} Publication.`
- **Corporate author:** `Organization Name. Year. \textit{Title.} Publication Type.`

**Required fields:** Every `\sourcecite{}` must contain all of:
1. **Author** (personal name as Last, First; or organization name)
2. **Year**
3. **Title** wrapped in `\textit{}`
4. **Publication** (journal, publisher, report type, or program name)

If any field is missing, the citation is incomplete. "BCG. 2025." alone is wrong because it has no title and no publication. The correct form is "BCG. 2025. \textit{AI Maturity Report.} Boston Consulting Group."

**Multi-source citations:** When a slide cites multiple sources, separate them with a semicolon and space. Each source must be a complete citation: `Last, First. Year. \textit{Title.} Publication; Last, First. Year. \textit{Title.} Publication.`

**Usage example:**

```latex
\begin{frame}{AI Adoption Outpaces Prior Technologies}
\begin{columns}[T]
\begin{column}{0.52\textwidth}
  \begin{itemize}
    \item ChatGPT reached 100M users in 2 months
    \item Enterprise adoption at 55\% within 18 months
  \end{itemize}
\end{column}
\begin{column}{0.45\textwidth}
  % TikZ chart here
\end{column}
\end{columns}
\sourcecite{Brynjolfsson, Erik, et al. 2024. \textit{Generative AI at Work.} NBER Working Paper.}
\end{frame}
```

### Rules for Citations and Notes

- Source citations use `\sourcecite{}` only. No custom footer macros, no raw "Source: X (year)" text, no manual `\vfill` constructions outside of `\sourcecite{}`.
- The `\sourcecite{}` macro renders text in upright `\scriptsize`. Titles must use `\textit{}` so they appear visually italicized within the otherwise upright citation text.
- Speaker notes use `\note{}` (invisible on slides). Never use visible `\insertnote` macros.
- Insertion or placement comments use LaTeX `%` comments during drafting, never visible text.
- No visible footer content (the footline template reserves space only).

### Bottom-of-Slide Text Hierarchy

Three tiers of bottom-of-slide text, each with distinct formatting. Never mix them:

| Text type | Formatting | Examples |
|---|---|---|
| **Source citation** | `\sourcecite{...}` (renders as `\scriptsize`, MedGray, right-aligned overlay) | Author. Year. *Title.* Publication. |
| **Facilitator note / content framing** | `\footnotesize\color{CharText}` | "Commit before looking around", "The case does not resolve this tension" |
| **Table/data footnote** | `\scriptsize\color{CharText}` | "*FY 2024 net income includes a one-time tax benefit" |

**Rule:** Never use `\scriptsize\color{MedGray}` outside of `\sourcecite{}`. That size/color combination is reserved for source attributions. Facilitator instructions, content framing notes, activity callouts, and discussion prompts use `\footnotesize\color{CharText}` so they are visually distinct from citations.

## Slide Types and Patterns

### 1. Dark Accent Slides (Title, Closing)

Override background to SlateNavy. Use `[plain]` frame option. All text in white, with DeepTeal and CyanBlue for secondary elements:

```latex
{
\setbeamercolor{background canvas}{bg=SlateNavy}
\begin{frame}[plain]
\vfill
\begin{center}
  {\color{white}\Large\bfseries Title Text}\\[14pt]
  {\color{DeepTeal!60!white}\normalsize Author or subtitle}\\[3pt]
  {\color{MedGray}\small Affiliation}\\[10pt]
  {\color{CyanBlue}\small Additional info}
\end{center}
\vfill
\end{frame}
}
```

Use this pattern for:
- **Title slide** (first slide)
- **Closing slide** (centered quote or statement)

### 1b. Key Takeaways Slide

White background with numbered takeaways. Use DeepTeal numbering and SlateNavy text:

```latex
\begin{frame}{Key Takeaways}
\begin{enumerate}
  \item[\textcolor{DeepTeal}{\bfseries 1.}] {\color{SlateNavy}\textbf{First takeaway.}} Supporting detail in CharText.
  \item[\textcolor{DeepTeal}{\bfseries 2.}] {\color{SlateNavy}\textbf{Second takeaway.}} Supporting detail in CharText.
  \item[\textcolor{DeepTeal}{\bfseries 3.}] {\color{SlateNavy}\textbf{Third takeaway.}} Supporting detail in CharText.
\end{enumerate}
\end{frame}
```

### 2. Two-Column Content Slides

The standard layout for most content slides. Use `\begin{columns}[T]` with a text column and a visual column:

```latex
\begin{frame}{Frame Title}
\begin{columns}[T]
\begin{column}{0.52\textwidth}
  % Text content, bullet points, equations
\end{column}
\begin{column}{0.45\textwidth}
  % TikZ diagram or pgfplots chart
\end{column}
\end{columns}
\end{frame}
```

Column width split is typically 50/47, 52/45, or 55/42 depending on content density.

**Chart sizing in columns:** When placing a pgfplots chart in a column, size conservatively. Leave at least 15% of the column width as horizontal buffer. For a `0.45\textwidth` column, use `width` no larger than `5.5cm`. For slides with both a chart and a `\sourcecite`, keep chart content in the upper 75% of the slide area. TikZ content should not extend below `y = -2.0` in a center-anchored tikzpicture on any slide that uses `\sourcecite`.

### 3. Full-Width Chart Slides

For large data visualizations that need the full slide width:

```latex
\begin{frame}{Chart Title}
\vskip2pt
\begin{center}
\begin{tikzpicture}[font=\footnotesize]
  \begin{axis}[width=12.5cm, height=6cm, ...]
    ...
  \end{axis}
\end{tikzpicture}
\end{center}
\end{frame}
```

### 4. Data Table Slides

Centered tables with `booktabs` rules and alternating `PaleBlue` row shading:

```latex
\begin{frame}{Key Finding from the Data}
\begin{center}
\footnotesize
\begin{tabular}{@{}l*{N}{>{\centering\arraybackslash}p{1.8cm}}@{}}
\toprule
...
\rowcolor{PaleBlue}
...
\bottomrule
\end{tabular}
\end{center}
\end{frame}
```

## Chart Styling (pgfplots)

### Standard Axis Configuration

```latex
\begin{axis}[
    width=12cm, height=6cm,          % Full-width charts
    xlabel={...},
    ylabel={...},
    xlabel style={font=\footnotesize},
    ylabel style={font=\footnotesize},
    tick label style={font=\scriptsize,color=CharText,/pgf/number format/1000 sep={}},
    axis lines=left,
    axis on top,
    legend style={at={(1.03,0.5)},anchor=west,font=\small,draw=none,fill=none},
]
```

**Axis line rules:**
- **Always use `axis lines=left`** to draw only the left y-axis and bottom x-axis. This produces clean, L-shaped axes without a box border. Never use the default box axes (all four sides), which create an enclosing rectangle that looks like unwanted grid lines.
- **Always use `axis on top`** to draw axis lines on top of plot content. This prevents bar edges from covering axis lines.
- **Never use `clip=false`.** The default `clip=true` prevents bars from extending outside the plot boundary. The combination of `axis lines=left` + `clip=true` produces contained bars with clean axes. If `nodes near coords` labels need headroom, increase `ymax` (or decrease `ymin` for negative values) to provide at least 15-20% beyond the tallest/deepest bar.

**Important:** Always include `/pgf/number format/1000 sep={}` in tick label style when the axis uses year values, to prevent comma formatting (e.g., `2,020` instead of `2020`).

### Color Assignments for Chart Series

| Series | Color | Line Style |
|--------|-------|------------|
| Primary / baseline | DeepTeal | `very thick` |
| Secondary / alternative | CyanBlue | `very thick` |
| Tertiary / positive | AccentGreen | `very thick` |
| Reference / trend line | MedGray | `dashed` |
| Fifth / tertiary | DustyPlum | `very thick` |
| Sixth / alert | AccentRed | `dashed, thick` |
| Dark / aggregate | SlateNavy | `very thick` |

### Bar Charts

**Never use `symbolic x coords` for bar charts.** `symbolic x coords` prevents axis line padding from working, making it impossible to ensure bars are contained within chart boundaries. Always use numerical coordinates with `xticklabels`:

```latex
ybar,
bar width=12pt,
xtick={0,1,2,3,4},
xticklabels={Label1,Label2,Label3,Label4,Label5},
xmin=-0.7, xmax=4.7,
axis lines=left,
axis on top,
ymin=0,
```

Data coordinates use integers: `(0,value) (1,value) (2,value)` instead of symbolic strings.

Use `fill=SlateNavy!80,draw=SlateNavy` for single-series bars. For dual bars, use `fill=CyanBlue!70,draw=CyanBlue` and `fill=DeepTeal!70,draw=DeepTeal`.

**Explicit `xmin`/`xmax` (required):** Always set explicit `xmin` and `xmax` to provide padding beyond the outermost bar positions. Do not use `enlarge x limits` as the sole containment mechanism; it is unreliable (does not work at all with `symbolic x coords`, and even with numerical coords can produce insufficient padding). For N categories at positions 0 through N-1:

| Categories | `xmin` | `xmax` | Notes |
|---|---|---|---|
| 3 | -0.7 | 2.7 | Standard padding |
| 4 | -0.7 | 3.7 | Standard padding |
| 5+ | -0.7 | (N-1)+0.7 | Standard padding |
| Grouped bars (2+ series) | -0.8 to -1.0 | (N-1)+0.8 to (N-1)+1.0 | Extra padding for wider bar groups |

**`ymin` and zero baselines:**
- **All-positive data:** Set `ymin=0` so the x-axis sits at zero. Do not add negative padding for data that is entirely positive.
- **Mixed positive/negative data:** Keep `ymin` at or below the lowest value. Add a zero baseline line:
  ```latex
  extra y ticks={0},
  extra y tick labels={},
  extra y tick style={grid=major, grid style={draw=CharText!50, thin}, tick style={draw=none}},
  ```

**Per-bar coloring with `bar shift=0pt` (required for single-bar-per-category charts):** When multiple `\addplot` commands are used so each bar gets a different color, and only one bar per category is visible, add `bar shift=0pt` to **each individual `\addplot`** command. This forces each bar to render centered on its x-coordinate, overriding pgfplots' automatic per-series offset. Without this, pgfplots treats each `\addplot` as a grouped series and shifts bars left/right of their labels. Use `ybar` (not `ybar=0pt`) on the axis; `ybar=0pt` only sets inter-bar gap to zero but does NOT disable the per-series positional offset.

```latex
% CORRECT: bar shift=0pt on each \addplot
\begin{axis}[ybar, bar width=22pt, ...]
  \addplot[fill=MedGray!60,draw=MedGray,bar shift=0pt] coordinates {(0,0.47)};
  \addplot[fill=CyanBlue!70,draw=CyanBlue,bar shift=0pt] coordinates {(1,0.50)};
  \addplot[fill=DeepTeal!80,draw=DeepTeal,bar shift=0pt] coordinates {(2,0.54)};
\end{axis}

% WRONG: ybar=0pt does NOT center bars
\begin{axis}[ybar=0pt, bar width=22pt, ...]
  \addplot[fill=MedGray!60,draw=MedGray] coordinates {(0,0.47) (1,0) (2,0)};
  ...
\end{axis}
```

Each `\addplot` should contain only the single coordinate for that bar (no zero-value placeholders at other positions).

**Bar width for grouped charts:** Reduce `bar width` proportionally to the number of series. A safe maximum: `bar width` should not exceed `(chart width / number of x-coords / number of series) * 0.6`.

**X-tick label rotation (required for narrow charts):** When a chart has 3 or more category labels and chart width is 6cm or less, default to `rotate=25, anchor=north east` on `xticklabel style`. Labels with spaces, long single words, and labels with special characters are especially prone to collision at narrow widths. Example:
```latex
xticklabel style={font=\scriptsize,rotate=25,anchor=north east},
```
For wider charts (> 6cm), horizontal labels are acceptable if the labels are short relative to the inter-tick spacing.

**No grids on bar charts:** Do not use `grid=major` as a general axis option on bar charts. The only exception is the zero baseline line drawn via `extra y ticks`, which uses `grid=major` scoped to that single tick.

### Stacked Bar Charts (`ybar stacked`)

For vertically stacked bar charts, use `ybar stacked` on the axis. Each `\addplot` adds a new layer to the stack (first = bottom, last = top).

**Legend order:** Always add `reverse legend` to the axis options so the vertical legend reads top-to-bottom matching the visual stack top-to-bottom. Without `reverse legend`, pgfplots lists legend entries in `\addplot` order (bottom stack layer = top legend entry), which reverses the visual correspondence. Do not use `reverse legend` for non-stacked bar charts, where the default order already matches.

```latex
\begin{axis}[
    ybar stacked,
    reverse legend,
    legend style={at={(1.03,0.5)},anchor=west,font=\small,draw=none,fill=none},
    ...
]
  \addplot[fill=MedGray!60,draw=MedGray] coordinates {(0,30) (1,25) (2,20)};   % bottom layer
  \addplot[fill=CyanBlue!70,draw=CyanBlue] coordinates {(0,40) (1,45) (2,50)};  % middle layer
  \addplot[fill=DeepTeal!80,draw=DeepTeal] coordinates {(0,30) (1,30) (2,30)};   % top layer
  \legend{Top Layer, Middle Layer, Bottom Layer}  % reads top-to-bottom matching visual stack
\end{axis}
```

**Legend position for stacked charts:** Place the legend outside the plot area (right side: `at={(1.03,0.5)},anchor=west`). Stacked bars frequently reach near `ymax`, making any inside-plot legend position risky for overlap.

**Axis containment verification:** After compiling, visually confirm that:
1. All bars are fully contained within the plot area (no bar extends outside the clipping boundary)
2. Only left and bottom axis lines are drawn (no box border, no right or top axis lines)
3. No visible grid lines unless explicitly intended (check for stray box borders from default axes)
4. Axis lines are drawn on top of bars (no bar edge covers an axis line)
5. All bars have the same rendered thickness (uniform width in vertical charts, uniform height in horizontal charts)
6. X-axis lines extend beyond the outermost bars on both sides (confirmed by `xmin`/`xmax` padding)

### Horizontal Bar Charts (`xbar`)

When creating horizontal bar charts where each bar represents a different category:

**Single-color bars:** Use a single `\addplot` with all coordinates:
```latex
\addplot[fill=SlateNavy!80,draw=SlateNavy]
  coordinates {(42,1) (56,2) (75,3)};
```

**Per-bar colors (common pitfall):** If you use multiple `\addplot` commands with one coordinate each at different y-values, pgfplots treats them as grouped bar series and offsets the bars, causing bar-label misalignment. Fix this with `bar shift=0pt` on each `\addplot`:

```latex
\addplot[fill=DeepTeal!70,draw=DeepTeal,bar shift=0pt] coordinates {(1200,1)};
\addplot[fill=CyanBlue!70,draw=CyanBlue,bar shift=0pt] coordinates {(9100,2)};
```

Alternatively, use a single `\addplot` with uniform color if distinct colors are not essential.

Always verify that y-axis tick labels (`yticklabels`) align vertically with their corresponding bars in the compiled PDF. If labels appear offset from their bars, the chart has a grouped-series misalignment.

### Grids

```latex
grid=major,
grid style={gray!20},
```

Use grids on time-series and line charts. Omit on bar charts, scatter plots, and conceptual diagrams. The only exception for bar charts is a zero baseline line drawn via `extra y ticks` (see Bar Charts section), which uses a scoped `grid=major` on that single tick.

### Chart Region Fills

When shading regions under curves or between areas on a chart:

```latex
\addplot[fill=DeepTeal,fill opacity=0.15,draw=none,domain=...] {...} \closedcycle;
\addplot[fill=CyanBlue,fill opacity=0.12,draw=none,domain=...] {...} \closedcycle;
```

Use opacity 0.12-0.15 for region fills on white backgrounds to ensure visibility.

### Annotations on Charts

- Data point labels: `\node[text=DeepTeal,font=\tiny\bfseries,above left]`
- Reference lines: `\draw[dashed,AccentRed,thick]`
- Callout annotations: `\node[text=CyanBlue,font=\scriptsize\bfseries]` with `\draw[-{Stealth},CyanBlue,thick]` arrow

**`nodes near coords` defaults:** Always include `/pgf/number format/fixed,/pgf/number format/precision=2` in the `nodes near coords style` when using `nodes near coords`. Without this, pgfplots renders small decimal values (e.g., 0.04) in scientific notation (`4 . 10^{-2}`), which is unreadable on slides.

```latex
nodes near coords,
nodes near coords style={font=\tiny\bfseries,/pgf/number format/fixed,/pgf/number format/precision=2},
```

**Value label collisions near axes:** When using `nodes near coords` on bar charts, short bars near the axis origin produce value labels that sit at the same height as y-axis tick labels, causing collisions. For the first and last bar groups in particular, verify that value labels do not overlap axis tick labels. If collisions occur, add `xshift` or `yshift` to the `nodes near coords` style, or use conditional placement to offset labels on short bars away from the axis.

### Chart Annotation Recipes

**Never place chart annotations as freestanding TikZ `\node` elements at hardcoded picture coordinates.** Hardcoded coordinates break when chart dimensions, axis ranges, or content changes. Always use pgfplots-native mechanisms.

**Reference line labels:** Use `extra y ticks` and `extra y tick labels`, never a separate `\node`:

```latex
\begin{axis}[
  % ... normal axis options ...
  extra y ticks={0.80},
  extra y tick labels={\color{AccentRed}\bfseries 0.80},
  extra y tick style={
    grid=major, grid style={dashed, AccentRed, thick},
    tick style={draw=none},
    ticklabel pos=right
  },
]
```

The label is part of the axis tick system, so it is guaranteed to align with the reference line and cannot overlap chart content inside the plot area.

**Annotation arrows on bar charts:** Always use `axis cs:` coordinates inside the `axis` environment. Before placing the annotation, compute the maximum bar/stack height and set `ymax` with headroom:

```latex
% For stacked bars with max stack height = 22:
% ymax >= max_stack + text_height + arrow_gap + margin
% ymax >= 22 + 5 + 2.5 + 2.5 = 32
\begin{axis}[ymax=32, ...]
  % ... bar plots ...
  % Annotation: text and arrow share the same x-coordinate for vertical alignment
  \node[font=\footnotesize,align=center] (zlabel) at (axis cs:1.5,28)
    {Annotation text};
  \draw[-{Stealth},AccentRed,thick] (axis cs:1.5,26) -- (axis cs:1.5,23);
\end{axis}
```

**Combined count + percentage labels:** When bars need both a count and a percentage, use `axis cs:` nodes inside the axis environment with colors matching their bars:

```latex
\node[above,font=\scriptsize\bfseries,text=DeepTeal] at (axis cs:0,133)
  {133 (74\%)};
```

## TikZ Diagram Styling

### Text Anchoring in Boxes

**All TikZ content boxes must use `anchor=north` with `align=left`.** Text inside boxes anchors from the top with left alignment, not centered vertically or horizontally. This ensures text starts at the same y-position regardless of content length, and reads naturally from top-left. Use `inner sep=6pt` (or `inner xsep`/`inner ysep` if different horizontal and vertical padding is needed) to provide appropriate margins between the text and the box edges.

The only exception is small single-line label nodes (e.g., header badges, axis labels, chart annotations) where centering is appropriate because the text is one short line.

For multi-line content boxes:
```latex
\node[..., anchor=north, align=left, inner sep=6pt, text width=3.5cm] at (x,y) {...};
```

For single-line labels (exception):
```latex
\node[..., align=center] {...};
```

### Concept/Framework Boxes

```latex
\node[draw=DeepTeal,fill=DeepTeal!15,rounded corners=2pt,
  minimum width=2.2cm,minimum height=0.6cm,anchor=north,align=left,inner sep=6pt] {...};
```

- Use `!15` fill tint on white backgrounds for visibility
- `rounded corners=2pt` for small nodes, `4pt` for large cards
- Draw color matches the fill base color

### Info/Detail Boxes

For bordered boxes with light fill and colored text (not full-color cards):

```latex
\node[draw=DeepTeal,fill=DeepTeal!15,rounded corners=3pt,inner sep=6pt,
  text width=0.93\columnwidth,anchor=north,align=left] {...};
```

Use `!12` to `!15` tint strength on white backgrounds.

### Emphasis/Info Cards (TikZ)

For solid-color question/info cards with white text:

```latex
\node[fill=DeepTeal,text=white,rounded corners=4pt,inner sep=10pt,
  text width=0.90\columnwidth,anchor=north,align=left,font=\normalsize\bfseries] {...};
```

Use DeepTeal for primary, CyanBlue for secondary, SlateNavy for method/summary.

### Scenario/Category Cards

For side-by-side comparison boxes with uniform dimensions, draw empty box nodes and overlay text separately. This guarantees all boxes render at exactly the same size regardless of content length.

```latex
% Define the box style (no text content — minimum dimensions ARE the rendered size)
\tikzset{cardbox/.style={draw=#1,fill=#1!15,rounded corners=4pt,
  minimum width=3.9cm,minimum height=4.2cm}}

% Draw empty boxes at uniform size
\node[cardbox=AccentGreen] (b1) at (0,0) {};
\node[cardbox=CyanBlue]    (b2) at (4.5,0) {};
\node[cardbox=DustyPlum]   (b3) at (9.0,0) {};

% Overlay text on each box (does not affect box dimensions)
\node[anchor=north west,text width=3.5cm,align=left,font=\small]
  at ([shift={(6pt,-6pt)}]b1.north west)
  {\textbf{\color{AccentGreen}Title}\\[4pt] Body text here...};
\node[anchor=north west,text width=3.5cm,align=left,font=\small]
  at ([shift={(6pt,-6pt)}]b2.north west)
  {\textbf{\color{CyanBlue}Title}\\[4pt] Body text here...};
\node[anchor=north west,text width=3.5cm,align=left,font=\small]
  at ([shift={(6pt,-6pt)}]b3.north west)
  {\textbf{\color{DustyPlum}Title}\\[4pt] Body text here...};
```

**Why empty boxes:** Putting text inside a `minimum height` node causes auto-expansion when content exceeds the minimum, producing non-uniform heights. Empty box nodes render at exactly their minimum dimensions. The text overlay is independent and does not affect box size.

**Critical: the overlay has no overflow protection.** The text overlay node is independent of the box. If the overlay content is taller than the box, it silently renders below the box edge. There is no LaTeX warning, no compilation error, and no visual clipping. You must verify content fits *before* writing it.

**Height computation procedure (mandatory before writing overlay content):**

1. Compute characters per line:
   ```
   chars_per_line = (text_width - leftmargini) / avg_char_width
   ```
   Average character widths: `\scriptsize` = 1.5mm, `\footnotesize` = 1.7mm, `\small` = 1.9mm

2. Write each bullet item to fit within that character count. If an item exceeds it, the item wraps and counts as 2+ lines.

3. Compute total content height:
   ```
   header:        ~3.5mm (\small\bfseries) or ~3.0mm (\footnotesize\bfseries)
   subtitle:      ~2.5mm (if present)
   gap after hdr: ~1.4mm (4pt \\[4pt])
   per text line: ~3.5mm (\footnotesize) or ~3.0mm (\scriptsize)
   itemsep:       ~0.7mm (2pt) between items
   minipage overhead: ~2mm
   top shift:     inner_sep value (typically 6pt = ~2.1mm)
   ```

4. Verify: `total_content_height < minimum_height - top_shift`

**Quick reference (max single-line items per box height, with header):**

| Font size | text width | Chars/line | 2.4cm box | 3.8cm box | 5.2cm box |
|-----------|-----------|------------|-----------|-----------|-----------|
| `\scriptsize` | 3.5cm | ~21 | 3 | 6 | 9 |
| `\footnotesize` | 4.6cm | ~25 | 3 | 5 | 8 |
| `\footnotesize` | 5.1cm | ~28 | 3 | 5 | 8 |
| `\small` | 5.5cm | ~28 | 2 | 5 | 7 |

Wrapping items count as 2 lines. If any item exceeds chars/line, either shorten the text or increase the box height.

Use `!15` tint on white backgrounds.

### Chain / Flow Diagram Nodes

For chain nodes that must be uniform size, use the empty-box-plus-overlay pattern:

```latex
\tikzset{chainbox/.style={draw=#1,fill=#1!18,rounded corners=2pt,
  minimum width=2.5cm,minimum height=0.5cm}}

% Draw empty uniform boxes
\node[chainbox=DeepTeal] (c1) at (0,0) {};
\node[chainbox=CyanBlue] (c2) at (3.2,0) {};
% Overlay text
\node[font=\small,text=DeepTeal] at (c1) {Step 1};
\node[font=\small,text=CyanBlue] at (c2) {Step 2};
```

For single-line chain nodes where content is guaranteed short (labels, step names), centering the text node at the box center is acceptable. Use `anchor=north west` with shift for multi-line content.

Use `!18` tint for chain/flow nodes on white backgrounds for clear differentiation.

### Sibling-Node Uniformity

When multiple TikZ nodes appear in a row, grid, or group (e.g., phase boxes, flow steps, comparison cards), all sibling nodes must render at uniform size.

**Width: use `text width`, not `minimum width`.** `minimum width` is a floor; content wider than the minimum causes auto-expansion, producing visually non-uniform boxes. `text width` is a fixed constraint that forces text wrapping, producing uniform width regardless of content. Always use `text width` for sibling nodes:

```latex
\tikzset{phase/.style={draw=DeepTeal,fill=DeepTeal!15,rounded corners=2pt,
  text width=3.8cm,minimum height=0.7cm,anchor=north,align=left,
  inner sep=6pt,font=\small}}
```

**Height:** `minimum height` is also a floor. Content with more lines will expand the box beyond the minimum. **Primary fix: use the empty-box-plus-overlay pattern** (see Scenario/Category Cards above). Draw empty box nodes at fixed dimensions (no text content, so the minimum IS the rendered size), then overlay text with separate `\node[anchor=north west]` elements. This guarantees uniform height regardless of content length. Alternative: use a booktabs table instead (tables enforce uniform row heights natively via `\arraystretch`). When the content is structured data (problem/fix pairs, feature lists, ranked items), prefer a table.

**Positioning: prefer relative over absolute.** When placing a sequence of nodes in a vertical or horizontal flow, use relative positioning (`below=10pt of`, `right=10pt of`) instead of absolute coordinates. Relative positioning produces uniform gaps regardless of content height. Absolute coordinates (`y=0, y=-1.5, y=-3.0`) produce inconsistent visual gaps when nodes have different content heights.

**When to use tables instead of TikZ boxes:** If the content is a list of items with parallel structure (e.g., five failure modes with fixes, three priorities with descriptions), a booktabs table with colored cells, `\rowcolor`, and `\arraystretch` is both more visually consistent and more maintainable than TikZ boxes. TikZ should be reserved for content with genuine spatial relationships (flows, cycles, hierarchies).

### Arrows

```latex
\draw[-{Stealth},DeepTeal,thick] (from) -- (to);
```

Use `Stealth` arrowheads from `arrows.meta`. Arrow color should match the associated element. Do not use MedGray or lighter palette colors for arrows, thin connecting lines, or text labels on white backgrounds; these are barely visible when projected in classrooms. Use DeepTeal, CyanBlue, or darker palette colors for connecting elements and text labels. MedGray is reserved for `\sourcecite{}` text and chart reference/trend lines only.

### Braces

```latex
\draw[thick,CyanBlue,decorate,decoration={brace,amplitude=5pt,mirror}]
  (start) -- (end) node[midway,right=6pt,text=CyanBlue] {...};
```

### Chain / Flow Diagrams

Nodes connected by `MedGray` arrows, with the weak/highlight node in `AccentRed`:

```latex

  \draw[-{Stealth},MedGray] (C\i) -- (C\the\numexpr\i+1\relax);

```

## Table Styling

- Use `booktabs` (`\toprule`, `\midrule`, `\bottomrule`) exclusively. No vertical rules.
- Alternating row shading: `\rowcolor{PaleBlue}` on every other data row.
- Font size: `\footnotesize` for table body.
- Column alignment: `@{}` to remove outer padding. Use `>{\centering\arraybackslash}p{1.8cm}` for numeric columns.
- Section dividers within tables: `\midrule` followed by `\multicolumn{N}{l}{\textit{Section label:}}`
- Row shading width fix: when `\rowcolor{PaleBlue}` extends wider than `\booktabs` rules, replace default column padding with explicit inter-column spacing: use `@{\hskip 8pt}` between columns in the tabular spec, set `\setlength{\tabcolsep}{0pt}` before the table, and add `\renewcommand{\arraystretch}{1.15}` for vertical breathing room between shaded rows.

## TikZ Generation Safety

### Never use `scale=` on complex tikzpictures

The TikZ `scale` option (e.g., `scale=0.8` on a `tikzpicture` environment) scales coordinates but NOT text. A diagram drawn at scale=1.0 with labels that fit their nodes will, at scale=0.8, have shrunken nodes with unchanged text that overflows its containers. Arrow endpoints also shift while label positions remain at their original coordinates, producing misalignment.

When a diagram is too large for the slide, reduce the diagram's internal dimensions (node sizes, spacing values, font sizes) rather than applying a global scale transform.

### Never define parameterized TikZ styles inside Beamer frames

Beamer captures frame bodies as macro arguments, consuming `#` characters before TikZ processes them. A style defined inside a frame like `\tikzset{mybox/.style={draw=#1}}` produces "Illegal parameter number" errors because Beamer's argument parser interprets `#1` as a frame argument reference.

**Fix:** Define all parameterized styles in the preamble with `\tikzset{}`. Non-parameterized styles (no `#`) can be defined anywhere.

```latex
% CORRECT: parameterized style in preamble
\tikzset{mybox/.style={draw=#1,fill=#1!15,rounded corners=2pt}}

\begin{document}
\begin{frame}
\begin{tikzpicture}
  \node[mybox=DeepTeal] {Content};  % works
\end{tikzpicture}
\end{frame}

% WRONG: parameterized style inside frame
\begin{frame}
\tikzset{mybox/.style={draw=#1}}  % "Illegal parameter number" error
\end{frame}
```

## Frame Overflow Management

**Never use `[shrink=N]` on Beamer frames.** The `shrink` option scales frame content inside a `\resizebox`, which distorts TikZ `remember picture,overlay` coordinate calculations. This causes `\sourcecite` citations and other overlay-positioned nodes to appear displaced (typically in the middle of the slide instead of at the bottom). The displacement scales with the shrink value and cannot be worked around with alternative overlay approaches.

When content overflows a frame, reduce the content rather than scaling the frame:

- Reduce font size: `\footnotesize` or `\scriptsize` for dense content
- Reduce vertical spacing: `\vskip2pt` instead of `\vskip4pt`, tighter `\itemsep`
- Reduce `\arraystretch` for tables (e.g., `\renewcommand{\arraystretch}{1.0}`)
- Reduce TikZ/pgfplots figure dimensions (`width`, `height`)
- Split content across two slides if reduction is insufficient

## Cognitive Density Guidelines

- **Slide titles are the key message.** Each title should concisely convey the slide's main point or finding (e.g., "Remote Workers Report 23% Higher Satisfaction" not "Survey Results"). Keep titles to one line when possible; two lines maximum.
- **Two-column slides:** text on left, visual on right (or vice versa). Never two text-heavy columns.
- **Tables and charts:** one major data element per slide. The title should state the interpretation or key finding.
- **Equations:** display mode (`\[ ... \]`) for important equations. Inline for minor expressions.
- **Dark slides** (SlateNavy background) are reserved for transitions: title and closing only. Key Takeaways uses a white background.
- **No callout boxes in text.** Do not use `\emphbox`, `\highlight{}`, or similar highlighted box macros inside bullet lists, enumerate environments, or running prose. `\highlight{}` is reserved for standalone labels in TikZ diagrams only. For inline emphasis in bullets, use `\textbf{\color{DeepTeal}...}`. Convey the key point through the slide title instead of inline callouts.
- **No text shadows or effects.** Never apply `shadow`, `drop shadow`, `blur shadow`, or any shadow/glow options to text nodes, frames, or TikZ elements. All text must be flat and clean, with no shadow=true and no text effects.
- **No inline author references in bullet text.** Never reference authors by name inside bullet points, prose, or slide body text (e.g., "Hendrycks et al. find that..."). Instead, present the finding as a factual statement and attribute the source using `\sourcecite{}` at the bottom of the slide. Inline author references break the expository flow and are unnecessary when the citation is visible.
- **No MedGray on visible content.** MedGray (`#B0AFA8`) is reserved for `\sourcecite{}` text and chart reference/trend lines only. Never use MedGray for body text, date labels, timeline year labels, scale annotations, TikZ node text, fill colors, or any element that needs to be readable when projected in a classroom. Use CharText (`#3A3A3A`) for all readable text. Use palette accent colors (DeepTeal, CyanBlue, DustyPlum, AccentGreen, WarmAmber) at `!12` tint for light fills instead of MedGray.
