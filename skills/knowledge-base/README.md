# Knowledge Base

A personal knowledge management pipeline for documents: drop files into an inbox, get renamed, summarized, indexed, and organized output.

## Problem

Research and reading accumulate faster than they can be organized. PDFs land in Downloads. Interesting articles get bookmarked and forgotten. When you need to find what you read about a topic six months ago, there is no reliable way to search it.

The standard workaround is a folder with vague names and files you cannot find. The other workaround is a note-taking app that captures fragments but not full documents. Neither gives you a searchable, citable record of what you have actually read.

## Approach

The knowledge base treats documents as first-class citizens. Each document gets:

1. **A citation-standard filename** (`YYYY-MM-DD LastName. Title.ext`) so files sort chronologically and are identifiable without opening them.
2. **A structured summary** (`_summary.md`) generated by a summary skill appropriate to the document type (academic or general). The summary is the primary reference artifact.
3. **A full-text extraction** (`_text.md`) for PDFs, enabling downstream search and slide generation without re-reading the source.
4. **An index entry** in a single markdown table (`index.md`) that fits in one context window read.

Documents are organized into topic folders you define. The skill does not impose a taxonomy; it discovers whatever folders exist and suggests assignments based on content.

## Operating Modes

**Process Inbox (`/kb`)**
Scan the inbox folder for new files. Each item is processed in its own subagent for context isolation. The parent reads the appropriate summary skill format before launching agents, then inlines those instructions into each agent prompt. After processing, presents a batch summary table with topic suggestions and waits for confirmation before moving files.

Supports three processing levels:
- Full: all pages, produces `_text.md` and `_summary.md`
- Triage: pages 1-4 only, produces `_summary.md`
- Lite: uses a companion `.txt` file you provide, produces `_summary.md` without reading the PDF

**Q&A (`/kb ask [question]`)**
Answers questions grounded in indexed sources. Reads the index, selects the 5-15 most relevant documents, reads their summaries, and synthesizes a cited answer. Distinguishes between what sources say, inference, gaps, and contradictions. Optionally saves answers back to the index.

**Move Project (`kb move <source_dir> <topic>`)**
Files the output of a content skill (slides, summary, etc.) that ran outside the knowledge base. Inventories artifacts, renames them to citation convention, determines whether to use flat or subfolder storage, moves everything, and updates the index.

## Design Rationale

**Agent-per-item architecture.** Each inbox item runs in its own subagent. This prevents content and image accumulation from hitting API request size limits on large batches. It also forces the parent to read and inline the correct summary format before launch, preventing improvised summaries that drift from the knowledge base's format standards over time.

**Citation-first naming.** The `YYYY-MM-DD Last. Title.ext` convention is borrowed from academic reference managers. Files sort by date, identify by author, and remain readable in a file browser without any app. The naming convention is the most durable part of the system.

**Two storage patterns.** Pattern A (per-document subfolder) is used when a document has been fully processed with slides and build artifacts. Pattern B (flat files) is the default for inbox-only processing. Slides skills can promote Pattern B items to Pattern A when they run. The index treats both identically.

**Append-only index.** During processing, the index only grows. Entries are never removed during a processing run, only during an explicit sync/health-check run that confirms source files still exist. This prevents accidental data loss when files are temporarily moved or a sync is interrupted.

**Blog posts as first-class knowledge.** The user's own writing is filed into topic folders alongside external sources, tagged with `[blog]` in the index. This makes it possible to ask the knowledge base "what have I written about X" alongside "what have I read about X."

**Topic folders are dynamic.** No hardcoded taxonomy. Any subdirectory (except reserved names like `aa-inbox/`, `aa-blog/`, `aa-recents/`) is a topic folder. Create a new folder and the next inbox run discovers it automatically.

## Usage

1. **Set up your knowledge base root.** Create a directory (e.g., `~/knowledge-base/`) with an `aa-inbox/` subfolder inside it. Update the path in SKILL.md to match.

2. **Configure your summary skills.** This skill calls out to an academic summary skill and a general summary skill. Adapt the references in SKILL.md to point to whichever summary skills you use. The format template from those skills is inlined into agent prompts at runtime.

3. **Drop files into `aa-inbox/`.** Any supported file type: PDF, markdown, DOCX, RTF, TXT, HTML, PNG, JPG, or a `.urls` file containing URLs to fetch.

4. **Run `/kb`.** The skill scans the inbox, processes each file, and presents a batch summary table with suggested topic assignments.

5. **Confirm or redirect topic assignments.** The skill moves files and updates the index after you confirm.

6. **Ask questions with `/kb ask [question]`.** The skill reads the index and relevant summaries, then synthesizes a cited answer.

7. **File pipeline output with `kb move`.** When a slides or summary skill runs on a document outside the knowledge base, use `kb move` to file the artifacts into the appropriate topic folder.

## Output

For each processed document:
- `YYYY-MM-DD Last. Title.ext` (renamed source file)
- `YYYY-MM-DD Last. Title_text.md` (full-text extraction, PDFs only)
- `YYYY-MM-DD Last. Title_summary.md` (structured summary)
- One row added to `index.md`
- Symlink in `aa-recents/` pointing to `_summary.md`

The index row format:
```
| Date | Author | Title | Topic | Summary |
| 2026-03-18 | Ipeirotis | Scalable Oral Assessments Using Voice AI | AI-articles | Voice AI oral exams at $0.42/student with multi-model deliberation achieving alpha=0.86 |
```
